MULTILINE PROMPT TEST FORMATS
=============================

METHOD 1: BLOCK DELIMITERS (RECOMMENDED)
=========================================
Use this format for long, complex prompts with multiple paragraphs:

---PROMPT-START---
You are an expert document analyst specializing in creating concise and informative summaries.
Your task is to create a comprehensive summary of the document text below.

CRITICAL SYSTEM INSTRUCTIONS:
1. You MUST respond with ONLY a valid JSON object - NO OTHER TEXT
2. Do NOT include any explanatory text, markdown, or other formatting

Please provide:
- A brief executive summary (2-3 sentences)
- Key findings or main points (bullet format)
- Any recommendations or action items
- Important dates, numbers, or metrics mentioned

Format your response as a JSON object with these fields:
{
  "executive_summary": "...",
  "key_findings": ["...", "..."],
  "recommendations": ["...", "..."],
  "metrics": {"key": "value"}
}
---PROMPT-END---
---DATA-START---
filename.pdf;application/pdf;H4sIAAAAAAAAA...your_base64_data_here...
---DATA-END---


METHOD 2: MULTILINE PREFIX
==========================
Use this for medium-length prompts:

prompt-multiline:You are an expert document analyst.
Please create a comprehensive analysis including:
1. Executive summary
2. Key findings  
3. Recommendations
4. Important metrics

Respond in JSON format only.

data:filename.pdf;application/pdf;H4sIAAAAAAAAA...your_base64_data_here...


METHOD 3: TRADITIONAL SINGLE LINE
=================================
Use this for short prompts (existing format still works):

prompt:Please analyze this document and provide a summary
data:filename.pdf;application/pdf;H4sIAAAAAAAAA...your_base64_data_here...


METHOD 4: NO PREFIX (SIMPLE)
============================
Just put your base64 data directly (uses default prompt):

filename.pdf;application/pdf;H4sIAAAAAAAAA...your_base64_data_here...


USAGE INSTRUCTIONS:
==================
1. Copy one of the formats above
2. Replace "H4sIAAAAAAAAA...your_base64_data_here..." with your actual base64 data
3. POST to: http://your-server:3600/api/v1/process-multiline
4. Set Content-Type: text/plain
5. Use the text as the request body

The API will now properly handle multiline prompts and return enhanced response with prompt_info!
